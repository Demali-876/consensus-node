1. Dedicated node role

This machine is being evaluated for registration as a dedicated node in the Consensus network. The network may use commissioned nodes to provide proxy, WebSocket, tunnel, routing, connectivity, and related infrastructure services.

By continuing setup, the operator acknowledges that this machine is intended to serve network requests on an ongoing basis and should not be registered casually, temporarily, or in a way that misrepresents its ability to operate.

2. Availability and uptime

If the node is accepted and commissioned, the operator is expected to keep it available 24/7, except for reasonable maintenance, outages outside the operator's control, or decommissioning approved by the network.

  • The node should remain powered on, connected, and reachable while commissioned.
  • The operator should avoid moving, throttling, firewalling, suspending, or repurposing the node in a way that makes it unreliable.
  • Repeated downtime, unstable connectivity, or poor performance may cause the node to be rejected, decommissioned, or discarded from the network.

3. Network requirements and public network use

IPv4 connectivity is required. IPv6 connectivity is optional and may be detected or used if available. A custom domain or network identifier may be issued for this node during registration.

The operator understands that the machine's public IP address, reachable ports, geographic region, bandwidth, latency, and availability may be used to benchmark, route, monitor, or serve network traffic.

4. Benchmarking and registration integrity

The operator agrees not to manipulate setup, benchmarking, network detection, wallet capture, registration, or monitoring results. The node should be evaluated under honest conditions that represent how it will actually operate after commissioning.

  • No artificial benchmark acceleration, traffic spoofing, uptime spoofing, location spoofing, or temporary network upgrades intended only to pass setup.
  • No registering machines the operator does not control or cannot responsibly maintain.
  • No changing code, environment variables, responses, or local state to falsify setup progress or registration eligibility.

5. Security and maintenance

The operator is responsible for keeping the machine reasonably secure, updated, and under their control. Private keys, wallet approvals, setup tokens, node credentials, and local state files should be protected from unauthorized access.

The operator should install updates, address security issues, and avoid running conflicting software that would compromise the node, expose credentials, or degrade network service.

6. Prohibited conduct

The operator agrees not to use the node or the Consensus network to perform, assist, hide, or monetize abusive behavior. Prohibited conduct includes fraud, credential theft, spam, malware distribution, unauthorized access, evasion of bans or rate limits, illegal content distribution, denial-of-service activity, and any attempt to harm the network or its users.

The operator also agrees not to sell, lease, transfer, or delegate control of the commissioned node in a way that breaks these obligations or makes the node unaccountable.

7. Traffic, logs, and operational data

The operator understands that the node may process network traffic as part of Consensus services. The operator should not inspect, retain, exploit, modify, or disclose traffic or operational data except as required for legitimate maintenance, security, debugging, or compliance with applicable obligations.

The network may collect operational signals such as uptime, latency, bandwidth, region, public IP reachability, version, benchmark results, and registration status to evaluate reliability and protect service quality.

8. Commissioning, removal, and rewards

Completing setup does not guarantee acceptance, commissioning, continued participation, rewards, or payment. The node may be rejected or removed if it does not meet network requirements, becomes unreliable, violates this agreement, or is no longer needed.

Wallet addresses collected during setup are used to identify payout destinations where applicable. The operator is responsible for connecting the correct wallets and maintaining control of those wallets.

9. Electronic signature and local record

Typing the confirmation phrase and electronic signature in the setup wizard records acceptance of this agreement in the local setup progress file. The record may include the accepted agreement version, signature text, and timestamp.

This setup agreement is an operational acknowledgement for node registration. Any separate written terms, policies, or legal requirements that apply to the Consensus network remain in effect.

10. Operator acknowledgement

By signing in the setup wizard, the operator confirms that they have read this agreement, understand the responsibilities of operating dedicated network infrastructure, and agree to operate the node honestly, securely, reliably, and in good faith.
